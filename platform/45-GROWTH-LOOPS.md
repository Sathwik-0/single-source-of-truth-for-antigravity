# Growth Loops

This document defines Chronicle's self-reinforcing organic growth loops.

---

## Loop 1: The Question Loop (The User Trust Loop)
As users interact with Chronicle, their successful queries reinforce their reliance on the system:

```text
               User Asks a Causal Question
                            │
                            ▼
          Receives Fast, Evidence-Backed Answer
                            │
                            ▼
           Citations Verified (Trust Increases)
                            │
                            ▼
              Asks More Complex Questions
```

---

## Loop 2: The Memory Loop (The Data Density Loop)
A denser database leads to more accurate contextual synthesis:

```text
         More Integrations Connected & Events Ingested
                            │
                            ▼
            Graph and Memory Density Increases
                            │
                            ▼
               Higher Precision Answers
                            │
                            ▼
            Increased Daily Active Usage (DAU)
```

---

## Loop 3: The Team Loop (The Organic Expansion Loop)
Chronicle spreads within organizations through developer collaboration:

```text
         Engineer Solves a Hard Legacy/Incident Bug
                            │
                            ▼
       Shares the Causal Timeline Link in Slack / PR
                            │
                            ▼
      Teammates and Managers View the Cited Evidence
                            │
                            ▼
     Team Adopts Chronicle to Preserve Workspace Memory
```

---

## Loop 4: The Repository Expansion Loop
Expanding scope within a single organization:

```text
           Connect One Initial Repository
                            │
                            ▼
        Experience Sub-30s Time To Understanding
                            │
                            ▼
     Connect Remaining Private Repositories & Org Slack
```
