# Pricing Models & Value Rules

This document outlines the commercial structures and monetization policies for Chronicle.

---

## 1. Monetization Plan

### Free Tier (Hobbyists & Single Repos)
* **Target:** Individual developers and open-source maintainers.
* **Pricing:** $0/month.
* **Constraints:**
  * 1 Connected GitHub repository.
  * Local-only memory storage.
  * Limit of 50 query planner searches per month.

### Pro Tier (Professional Developers)
* **Target:** Individual contributors and freelance engineers.
* **Pricing:** $29/month per user (Billed monthly).
* **Features:**
  * Unlimited public and private repositories.
  * Standard Timeline and Graph Engines.
  * Extended context compression support.

### Team Tier (High-Velocity Teams)
* **Target:** Engineering teams of 5 to 50 developers.
* **Pricing:** $299/month flat rate (Up to 10 users, +$15/user/month additional).
* **Features:**
  * Complete Slack integration (channels, threads, reactions).
  * Shared organizational memory and decision graphs.
  * Collaborative timeline sharing.

### Enterprise Tier (Corporate Security)
* **Target:** Scale-ups and mature tech enterprises.
* **Pricing:** Custom Annual Contracts.
* **Features:**
  * Single Sign-On (SSO) and advanced RBAC.
  * SOC2 compliance audit logs.
  * Custom data retention policies.
  * Dedicated cloud tenancy or private VPC deployment.

---

## 2. Core Value Rule: No Token-Based Billing

> **"Never charge based purely on tokens. Charge for value."**

Chronicle rejects standard utility-billing models based on raw input/output tokens:
* **The Problem:** Token-based pricing introduces budget unpredictability, forcing developers to hesitate before querying the system out of fear of unexpected charges.
* **The Rule:** Chronicle charges strictly for **value delivered**—defined as the complexity of the team’s organizational memory graph, the number of connected repositories, and collaborative sharing features. Query volume and token metrics are absorbed as infrastructure costs, ensuring developers can instinctively query Chronicle without friction.
