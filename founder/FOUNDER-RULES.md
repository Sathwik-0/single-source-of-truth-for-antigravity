# Chronicle Founder Rules

## Product Identity

Chronicle is an Engineering Knowledge System.

Its purpose is not to retrieve files.

Its purpose is to reconstruct understanding.

---

## North Star

Time To Understanding

---

## Core Values

Trust > Features

Evidence > AI

Memory > Retrieval

Understanding > Search

---

## Product Experience

The experience should feel like:

Perplexity + Linear + GitHub + Cursor

Characteristics:

- Fast
- Beautiful
- Keyboard-driven
- Evidence-first

Engineers should instinctively ask Chronicle questions.

---

## Product Philosophy

Chronicle reconstructs:

- Decisions
- Incidents
- Context
- Causality

Not documents.

---

## Builder Rules

If requirements are unclear:

STOP.

Ask founder.

Never invent behavior.

---

## Quality Rules

Forbidden:

- Fake buttons.
- Placeholder integrations.
- Dead screens.
- Decorative dashboards.

Every interaction must map:

UI
↓
API
↓
Service
↓
Database
↓
Response
