# Hallucination Prevention

This document defines the strict validation workflows, system instructions, and fallback rules that enforce Chronicle's "Evidence > AI" core value.

---

## 1. Strict Grounding Constraint

The primary system instruction for the synthesis model forces strict fact-containment:

> "You are the Chronicle Synthesis Engine. You answer engineering queries using ONLY the facts provided in the retrieved Context. If the context does not contain direct evidence to support a statement, you must not make it. If the context is insufficient to answer the query, state: 'I cannot find evidence to answer this question based on current records.'"

---

## 2. 1-to-1 Citation Verification Loop

Every response generated by the model must pass through a post-generation verification parser:

```text
       Generated Response Text (containing citations [Evidence-1], [Evidence-2])
                                  ↓
                       [Verification Parser]
                                  ↓
      1. Extract all statements associated with a citation.
      2. Verify that the statement matches the raw content of the source item.
      3. Verify that the citation link points to a valid file, commit, or message.
                                  ↓
                     { Any Statement Unverified? }
                       ┌───────────┴───────────┐
                       ▼                       ▼
                    [ Yes ]                 [ No ]
                       │                       │
         [Regenerate / Truncate statement]  [Stream to User]
```

If the verification fails, the specific ungrounded paragraph is removed or updated to say: *"Statement removed: insufficient evidence in logs."*

---

## 3. Safe Fallback Triggers

Chronicle will trigger an immediate fallback response instead of generating a synthesized answer if:
* **No high-scoring evidence:** No context chunks score above the `0.45` relevance threshold in the Evidence Ranking stage.
* **Conflicting temporal records:** The question planner identifies contradictory timeline facts that cannot be resolved without human input.
* **Ambiguous terms:** Multiple code symbols share the same name, and the user did not specify the repository scope.

By defaulting to safe fallback states, Chronicle maintains absolute trust as an engineering source of truth.
